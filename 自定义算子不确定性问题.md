导致不确定性的原因分析在给定的 Deformable DETR 的反向传播 CUDA 算子（col2im）中，引入无法通过随机种子固定的不确定性的根本原因在于浮点数的原子加法（atomicAdd）机制。
浮点数加法不满足结合律：在计算机底层，由于浮点数存在精度限制（截断误差），不同顺序的累加会产生微小的结果差异，
即 (a + b) + c != a + (b + c)
硬件级线程调度的随机性：在 CUDA 中，不同的 Query 可能会采样到特征图上的同一个空间位置（即落入同一个 grad_value 的坐标内）。这使得多个不同的 CUDA 线程会同时对全局显存 grad_value 发起 atomicAdd 进行梯度累加。
而 GPU 硬件层面线程调度、Warp 到达内存单元的先后顺序是完全不可预测的。随机种子无效：随机种子（Seed）只能固定伪随机数生成器的输出，而 atomicAdd 的执行顺序属于 GPU 硬件底层的并发竞争行为，无法被任何软件层面的种子所控制。
这种微小的浮点舍入误差在深层网络的多次迭代反向传播中会被不断放大，最终导致模型训练结果无法严格复现。
除此之外，在 fallback 分支的代码（如 _gm 内核以及多 block 规约的内核）中，对 grad_sampling_loc 和 grad_attn_weight 也直接应用了浮点数 atomicAdd，同样会引入不确定性。

解决思路：
定点数累加技巧 (Fixed-Point Accumulation)
为了在保证高效率且不改变外部 C++ 接口的前提下解决这个问题，最权威且被广泛验证的做法是引入整数定点数累加：分配整型缓冲区：我们在算子内部临时分配具有同样大小的 long long（64位整型）张量，并初始化为 0。
缩放浮点数：在 CUDA 内核中，我们将原本要累加的浮点数梯度乘以一个极大的缩放因子，将其转换为 long long 类型的整数。整数原子相加：利用 unsigned long long 的 atomicAdd 进行累加。
由于整数加法完美满足结合律，即使线程执行顺序混乱，最终累加出的整型数值也绝对唯一（负数利用补码特性同样计算无误）。
后处理还原：内核执行完毕后，启动一个轻量级的 O(N) 并行内核，将累加好的整数除以相同的缩放因子，还原为浮点数并写入原始指针中。

这种方式的时间开销极小（通常 <0.1 毫秒），
完美兼顾了确定性与性能。
